# アーキテクチャ設計

このドキュメントは、スマホ完結型AI開発環境のアーキテクチャについて詳述します。

## 1. 設計思想

この環境は、以下の3つの原則に基づいています：

1.  **司令塔と実行部隊の分離**: Manusを「司令塔」としてタスク管理に専念させ、実際の開発作業はClaude Code等の外部AI CLI「実行部隊」に委譲します。
2.  **コスト最適化**: タスクの性質に応じて最適なAIを自動選択し、特に簡単なタスクはGeminiの無料枠を活用することで、コストを最小限に抑えます。
3.  **高品質な成果物**: Claude Code Skills（ガイドライン）を活用し、AIの出力を特定の品質基準に準拠させることで、高品質なコードとドキュメントを生成します。

## 3. 主要コンポーネント

### 3.1. `scripts/delegate.py`

この環境の中核をなすタスク委譲スクリプトです。

-   **役割**: Manusから受け取ったプロンプトを分析し、最適なAI CLIにタスクを振り分けます。
-   **機能**:
    -   **タスク分析**: プロンプトのキーワードを基に、タスクタイプ（開発、質問、レビュー等）を判定します。
    -   **AI選択**: タスクタイプに応じて、最もコスト効率の良いAI（Claude, Gemini, Codex）を選択します。
    -   **スキル適用**: タスクタイプに最適なスキル（ガイドライン）を自動でプロンプトに注入します。
    -   **フォールバック**: あるAIが失敗した場合、自動的に次の候補に切り替えて再試行します。
    -   **エラーハンドリング**: 各CLIのタイムアウトや認証エラーを捕捉し、分かりやすいエラーメッセージを返します。

### 3.2. `scripts/dev.py`

`delegate.py`をラップし、開発ワークフロー全体を統合するスクリプトです。

-   **役割**: 開発タスクの実行からGit操作、完了通知までを一気通貫で行います。
-   **機能**:
    -   **Git連携**: 新しいブランチの作成、変更のコミット、リモートリポジトリへのプッシュを自動化します。
    -   **LINE通知**: `notify.py`を呼び出し、タスクの完了をLINEに通知します。
    -   **時間計測**: タスクの実行時間を計測し、レポートに含めます。

### 3.3. `scripts/notify.py`

LINEへの通知を専門に扱うスクリプトです。

-   **役割**: Manus MCPのLINEサーバーを経由して、テキストメッセージまたはFlex Messageを送信します。
-   **機能**:
    -   **メッセージフォーマット**: 成功、失敗などのステータスに応じて、色やアイコンを使い分けたリッチな通知を作成します。
    -   **Flex Message**: 開発レポートなど、構造化された情報を送信するためにFlex Messageを使用します。

### 3.4. `skills/` ディレクトリ

Claude Codeの出力を制御するための「スキル」（ガイドライン）が格納されています。

-   **役割**: AIに対して、コーディング規約、設計思想、ベストプラクティスなどを具体的に指示します。
-   **例**:
    -   `frontend-design`: モダンなUIデザインの原則を定義します。
    -   `react-best-practices`: Reactコンポーネントのパフォーマンスや保守性に関する45以上のルールを定義します。
    -   `systematic-debugging`: 体系的なデバッグプロセスを定義します。

## 4. ワークフロー

### 4.1. 新機能開発ワークフロー

1.  **ユーザー**: Manusに「新機能を実装して」と指示します。
2.  **Manus**: `ai-dev-environment`をクローンし、`dev.py`を呼び出します。
    ```bash
    python3 scripts/dev.py "新機能を実装" --branch feature/new --commit --push --notify
    ```
3.  **`dev.py`**: Gitブランチを作成し、`delegate.py`を呼び出します。
4.  **`delegate.py`**: タスクタイプを「開発」と判定し、AIとして「Claude Code」を選択します。関連スキル（`frontend-design`等）をプロンプトに注入します。
5.  **Claude Code**: スキルに従って高品質なコードを生成し、ファイルを変更します。
6.  **`dev.py`**: 変更を検知し、Gitコミットとプッシュを実行します。
7.  **`dev.py`**: `notify.py`を呼び出し、LINEに完了通知を送信します。
8.  **Manus**: 結果を確認し、ユーザーに報告します。

### 4.2. 簡単な質問ワークフロー

1.  **ユーザー**: Manusに「Pythonのリストソート方法は？」と質問します。
2.  **Manus**: `delegate.py`を呼び出します。
    ```bash
    python3 scripts/delegate.py auto "Pythonのリストソート方法は？"
    ```
3.  **`delegate.py`**: タスクタイプを「質問」と判定し、AIとして「Gemini」を選択します（無料枠活用）。
4.  **Gemini**: 質問に回答します。
5.  **Manus**: 回答をユーザーに提示します。

## 5. 今後の拡張性

-   **新しいAIの追加**: `delegate.py`に新しいCLI呼び出し関数を追加するだけで、他のAIも容易に組み込めます。
-   **新しいスキルの追加**: `skills/`ディレクトリに新しいMarkdownファイルを追加するだけで、AIの能力を拡張できます。
-   **他の通知手段**: `notify.py`を修正すれば、SlackやEmailなど他の通知チャネルにも対応可能です。
