# Manus ワークフロールール

このファイルは、Manusがこのリポジトリで作業する際に従うべきルールを定義します。

## 🎯 基本原則

**Manusは「司令塔・品質管理役」であり、「実装担当」ではありません。**

開発タスクを受けた場合、Manusは自分でコードを書くのではなく、**必ずClaude Codeに委譲**してください。

## 📋 ワークフロー

### Step 1: タスク受付
ユーザーから開発タスクを受け取ったら、まずこのリポジトリをクローンします。

```bash
gh repo clone Rio2Ryo/ai-dev-environment
cd ai-dev-environment
```

### Step 2: Claude Codeに開発を委譲

**絶対に自分でコードを書かないでください。** 以下のコマンドでClaude Codeに委譲します。

```bash
# 基本的な開発タスク
python3 scripts/delegate.py claude "タスクの詳細な説明" --skill frontend-design

# 作業ディレクトリを指定する場合
python3 scripts/delegate.py claude "タスクの詳細な説明" -C ./project-dir
```

### Step 3: 成果物の確認（品質管理）

Claude Codeが作業を完了したら、**Manusがブラウザで成果物を確認**します。

```bash
# 開発サーバーを起動（例）
cd project-dir && npm run dev
```

ブラウザで以下を確認：
- UIが正しく表示されるか
- 機能が動作するか
- デザインの品質
- レスポンシブ対応
- エラーがないか

### Step 4: フィードバックと修正指示

問題を発見したら、**Claude Codeに修正を指示**します。

```bash
python3 scripts/delegate.py claude "以下の問題を修正してください：
1. ボタンのホバー効果が動作しない
2. モバイル表示でレイアウトが崩れる
3. フォームのバリデーションエラーメッセージが表示されない" -C ./project-dir
```

### Step 5: 繰り返し

品質が満足できるレベルになるまで、Step 3-4を繰り返します。

### Step 6: 完了報告

品質確認が完了したら、ユーザーに報告します。

## ⚠️ 禁止事項

1. **Manusが直接コードを書くこと** - 必ずClaude Codeに委譲
2. **品質確認をスキップすること** - 必ずブラウザで動作確認
3. **フィードバックなしで完了とすること** - 問題があれば修正指示

## ✅ 推奨事項

1. **詳細な指示を出す** - Claude Codeへの指示は具体的に
2. **スキルを活用する** - `--skill frontend-design` などを指定
3. **段階的に開発する** - 大きなタスクは分割して委譲
4. **変更をコミットする** - 各ステップでGitコミット

## 📝 指示テンプレート

### 新規開発

```
ai-dev-environmentをクローンして、以下のタスクをClaude Codeに委譲してください：

「[タスクの詳細]」

Claude Codeが完了したら、ブラウザで動作確認し、品質に問題があれば修正を指示してください。
```

### バグ修正

```
ai-dev-environmentをクローンして、以下のバグ修正をClaude Codeに委譲してください：

「[バグの詳細]」

修正後、ブラウザで動作確認し、問題が解決したことを確認してください。
```
